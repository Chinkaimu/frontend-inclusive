(1). Change the state, and the component will be rerendered.

(2). !!!When we call setState, the states can't be updated immediately, React.js cache the state in a queue((TODO: Is it the event queue ?)) and update the states later. If you want use the last state, function 'setState' provide the usage to pase a function with the param previous state value.

7. Lists and Keys

(1). React.js can render every elements in an array.

(2). !!Keys in array elements is important. Check the reason, diff algorithm compare the component with same id, if there are no change, elements only need to move or do nothing other than delete DOM and create DOM.


8. Life cycle

(1). mount lifecycle: constructor() => componentWillMount() => render() => insert DOM to page => componentDidMount() => componentWillUnmount

(2). update lifecycle: willReceiveProps()/setState() => shouldComponentUpdate() => componentWillUpdate() => render() =>componentDidUpdate()


9. ref

Property 'ref' is a function that's param is the DOM node. Developers can parse the DOM node to "this", and call it after node mounted.


10. children

All the components inserted in the component A could be got by 'props.children' in A.
